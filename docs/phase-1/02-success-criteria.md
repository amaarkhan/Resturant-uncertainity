# Phase 1 Success Criteria and KPI Targets

## Purpose
Define measurable targets so MVP and PMF conclusions are objective.

## A. MVP Build Completion Criteria
1. End-to-end flow works
- Input -> Processing -> Output -> Feedback saved

2. Stability criteria
- Core flow success rate >= 95 percent during demo rehearsal
- No blocker bug in recommendation flow

3. Delivery criteria
- Customer app deployed
- Admin panel deployed
- Both accessible via live URL

## B. MVP Validation Metrics (Operational)
1. Recommendation View Rate
- Definition: percent of active users opening recommendation page before prep
- Initial target: >= 60 percent

2. Recommendation Follow Rate
- Definition: percent of viewed recommendations acted upon
- Initial target: >= 50 percent

3. Feedback Completion Rate
- Definition: percent of active days with feedback submitted
- Initial target: >= 50 percent

4. System Reliability
- Definition: successful recommendation API responses / total requests
- Initial target: >= 98 percent

## C. PMF Direction Metrics (Pilot)
1. Waste Reduction
- Definition: percentage decrease in leftover waste vs baseline
- Threshold: >= 10 percent improvement

2. Stockout Reduction
- Definition: percentage decrease in stockout incidents vs baseline
- Threshold: >= 15 percent improvement

3. 4-Week Retention
- Definition: percent of pilot users active in week 4
- Threshold: >= 40 percent

4. Trust Score
- Definition: average user confidence score (1-5)
- Threshold: >= 4.0

5. Willingness to Pay Signal
- Definition: pilot users saying yes to paid continuation or converting from trial
- Threshold: >= 20 percent

## D. Decision Rules
- PMF Direction Positive: meet at least 4 of 5 PMF thresholds including Waste Reduction.
- Promising Not Yet PMF: meet 2-3 thresholds with clear upward trend.
- Pivot Needed: meet 0-1 thresholds or poor trust/retention despite usage.

## Phase 1 Signoff Condition
All KPI definitions, formulas, and thresholds are accepted before Phase 2 development begins.
